# PE-10 — Internationalisation (i18n)

> **Status:** Complete (2026-07-16). Bilingual storefront and admin via `next-intl`.

## Locales

| URL prefix | Language | Intl locale |
|------------|----------|-------------|
| `/en/` | Australian English | `en-AU` |
| `/zh/` | Simplified Chinese | `zh-CN` |

Configuration: `frontend/src/i18n/routing.ts` (`localePrefix: always`).

## Message catalogues

- `frontend/messages/en.json`
- `frontend/messages/zh.json`

Namespaces include customer shell (`home`, `nav`, `cart`, …) and admin forms (`adminProducts`, `adminOrders`, `adminPricing`).

## Navigation

- `LocaleSwitcher` in header
- Locale-aware routing: `@/i18n/navigation` (`Link`, `useRouter`, `usePathname`)

## SEO — hreflang

`generateMetadata` in `[locale]/layout.tsx` reads `x-next-pathname` from middleware and sets:

- `alternates.canonical`
- `alternates.languages` (`en-AU`, `zh-CN`, `x-default`)

Helper: `frontend/src/lib/seo.ts` — `buildHreflangAlternates(path, locale)`.

Set production base URL:

```env
NEXT_PUBLIC_SITE_URL=https://your-domain.com
```

## Middleware

`frontend/src/middleware.ts` combines:

1. Auth guards (protected routes)
2. `next-intl` locale routing
3. `x-next-pathname` header for per-path hreflang

## Boundaries

- Product/category **names** from API remain as stored (not auto-translated).
- Currency display uses AUD; `formatPrice` respects locale number formatting.
- Analytics/system admin pages may retain some English technical labels.
